PKCS#7 / CMS

CMS es un formato estándar de firma binaria estándar definido en la RFC 3852 del Internet Engineering Task Force (IETF). Este estándar define una sintaxis de encapsulación capaz de contener diversos tipos de contenidos, entre los que se encuentran los datos firmados (SignedData). El estándar CMS deriva de la especificación PKCS#7, define en la RFC 2315.

La versión de CMS generada por el Cliente @firma mantiene la compatibilidad con las firmas generadas a partir de la especificación PKCS#7.

La especificación de CMS / PKCS#7 define que los datos firmados son un atributo opcional dentro de los objetos de firma y no es necesario mantener una referencia a ellos. Esto da lugar a las firmas en modo implícito (firmas con los datos incrustados) y explícito (firmas sin los datos firmados). Que la firma no contenga los datos firmados no implica que no se pueda comprobar que la firma se realizó sobre unos datos concretos que se especifican de forma externa.

La Interfaz de Escritorio del Cliente @firma no permite generar firmas en este formato. Como alternativa, sí permite generarlas en el formato de firma avanzada CAdES.
